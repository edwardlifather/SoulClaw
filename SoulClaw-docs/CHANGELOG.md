# SoulClaw Release History

## v0.1.0 (2026-04-05) - The "SoulClaw" Rebirth
*The project reached its professional-grade milestone and underwent global branding.*

- **✨ Global Rebranding**: Officially renamed from MinGate to **SoulClaw**.
- **🌍 Internationalization**: 100% English UI for the management console.
- **🛠️ Toolset Expansion**: Ported core "Claw" tools from the OpenClaw repository:
  - **FileSystem**: Search, read, and write local files.
  - **Shell**: Direct terminal command execution.
  - **HTTP Fetch**: Real-time URL data pulling.
- **📚 Documentation**: Released full English Architecture and Design whitepapers.

## v0.0.5 (2026-04-04) - Intelligence & UX Refinement
*Focus on observability and smarter cognitive management.*

- **🚀 Live Streaming**: Implemented "Typewriter" effect for real-time AI replies.
- **👁️ Observability**: Added raw API JSON request/response view and Token usage tracking in UI.
- **🧠 Memory Self-Correction**: Implemented logic to automatically merge/correct conflicting long-term facts.
- **🏘️ Hybrid Network**: Added non-blocking **Polling Mode** fallback for Telegram (no more Ngrok required for local dev).

## v0.0.4 (2026-04-01 to 2026-04-03) - Stabilization
*Hardened the communication bridge and memory logic.*

- **💾 Transcript Persistence**: Switched to structured JSONL for reliable session logging.
- **🏗️ Three-Tier Memory Model**: Refined Fact/Preference/Project file segregation in Markdown.
- **🔐 Security**: Implemented white-list (allowFrom) and HMAC signature verification for Feishu/Telegram.

## v0.0.3 (2026-03-30) - Memory Foundations
*Implemented the "Recollection" approach over traditional Vector searches.*

- **🔍 Search Logic**: Developed keyword-based Markdown snippet retrieval.
- **📝 Soul Registry**: Created `soul.md` persona injection mechanism.

## v0.0.2 (2026-03-27 to 2026-03-29) - Channel Connect
*Established the first automated bridges.*

- **🤖 Telegram Adapter**: Initial support for Grammy-based messaging.
- **🏢 Feishu/Lark Adapter**: Webhook-based integration for enterprise productivity.

## v0.0.1 (2026-03-26) - The Initial Spark
*The concept was born to create a lean, single-user successor to the OpenClaw framework.*

- **🐣 Framework Generation**: Scaffolding created based on OpenClaw's minimal architectural concepts.
- **🎯 Mission Defined**: Target 95% code reduction while maintaining professional agentic capabilities.
